B: Machine-Centric Grid

Each machine gets its own card showing status, current job, part progress, and next-up. Red border on machines that are down. Designed for a TV on the floor — glanceable from across the room.

Command Center (A)

  • More information density — KPIs, queues, bottlenecks all visible
  • Better for managers sitting at a desk
  • Shows the big picture across all dimensions

Machine Grid (B)

  • More glanceable from across the shop floor
  • Operators immediately see their machine's status
  • Scales naturally as you add machines
  • Problems (red cards) jump out visually

Both can auto-refresh every 15-30 seconds. We can also build both as tabs on the same page.
