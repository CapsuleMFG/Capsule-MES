Scheduling Board Layout

How should the boss see and rearrange jobs across your 13 machines?

A

Swim Lane Board (Kanban-style)

Each machine is a vertical column. Jobs are cards stacked in priority order. Drag cards up/down to reprioritize, drag left/right to move between machines. Like Trello but for your shop floor.

Pros

  • Intuitive drag & drop
  • See all machines at once
  • Easy to move jobs between machines

Cons

  • 13 columns is a lot — needs horizontal scroll
  • Can group by machine type to reduce columns
B

Timeline / Gantt View

Each machine is a horizontal row. Jobs are blocks on a timeline. Drag to reschedule in time, drag between rows to reassign machines. Shows when things overlap or have gaps.

Pros

  • Shows time dimension — when things start/end
  • Easy to spot gaps and overlaps
  • Familiar Gantt chart concept

Cons

  • More complex to build (2 weeks is tight)
  • Requires estimated durations for each step
  • Harder to just "reorder priority" without time data
C

Priority List + Machine Assignment

One master priority list on the left (drag to reorder). Click a job to see its route steps. Assign each step to a machine. Each machine has a "Queue" tab showing its ordered work. Simpler but powerful.

  • Less visual than swim lanes
